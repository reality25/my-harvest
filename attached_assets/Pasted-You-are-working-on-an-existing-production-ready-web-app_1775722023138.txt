You are working on an existing production-ready web application called “Harvest,” which is already deployed on Vercel. The app is built using React 18, TypeScript, and Vite, with Tailwind CSS and shadcn/ui for UI components. Routing is handled by React Router v6, and server state is managed using TanStack React Query. Authentication and database functionality are already implemented using Supabase, including Google OAuth. The app already includes community features, a marketplace, and a basic AI service layer.

Your task is to significantly extend this application into a fully-featured, AI-powered farm management system that supports both crop farming and livestock farming. The system must feel intelligent, adaptive, and localized, and should integrate seamlessly into the existing architecture without breaking current features.

Start by building an adaptive onboarding system that captures detailed farm information when a user signs up or updates their profile. This onboarding must collect the farm type (crop farming, livestock farming, mixed farming, or niche farming such as beekeeping or aquaculture), specific crops or animal types, farm size, and precise location. This data must be stored in Supabase and used to dynamically configure the user experience. The UI, dashboard, and available tools should adapt based on this data, so that a crop farmer, livestock farmer, and beekeeper all see different relevant features without requiring separate apps.

Extend the database schema in Supabase to support a flexible and modular farm profile system. Create tables to store farms, farm entities (crops or animals), health records, activity logs, tasks, AI scans, alerts, and plans. The schema must be designed in a way that allows new farm types to be added in the future without requiring major structural changes. For example, a “farm_entities” table should be able to represent both crops and animals with type-based differentiation.

Implement a comprehensive farm management module that allows users to track crops and animals over time. For crops, include growth stages, planting dates, fertilizer usage, irrigation, and disease history. For livestock, include health records, feeding schedules, vaccinations, and production metrics such as milk yield or egg count. All data should be tied to the user and farm, and should be easily retrievable and editable through the UI.

Build a smart task and reminder system that acts as a farm scheduler. The system should allow both manual task creation and AI-generated tasks. Tasks should include properties such as type (crop or livestock), title, description, due date, priority, and whether the task was generated by AI. The AI should generate tasks based on farm data, such as upcoming growth stages, vaccination schedules, or weather conditions. These tasks should be displayed in a clean, mobile-friendly dashboard and should support notifications (initially in-app, with future support for push notifications).

Enhance the existing AI service layer to support both plant and animal intelligence. Integrate Hugging Face models (either via API or self-hosted endpoints) to enable plant disease detection through image uploads. Users should be able to upload or capture images, which are then sent to an AI endpoint for analysis. The response should include disease name and confidence level, and must be transformed into clear, actionable advice for farmers, including treatment steps and prevention tips. Store all scan results in Supabase for historical tracking and insights.

For livestock, implement a symptom-based AI advisory system where users input observed symptoms, and the AI returns possible conditions along with recommended actions. Ensure that all AI outputs are presented in simple, practical language and include a confidence indicator and a disclaimer encouraging expert consultation when necessary.

Integrate a weather intelligence system that uses a reliable weather API to fetch location-based data such as temperature, rainfall, and humidity. Based on this data, the AI should generate contextual farming advice, such as adjusting irrigation schedules, delaying fertilizer application, or warning about increased risk of pests or disease. The system must also support alerts for extreme weather events such as floods or droughts, and provide actionable recommendations in those cases.

Build a localization engine that ensures all insights, alerts, and recommendations are tailored to the user’s geographic region. This includes pest outbreaks, disease warnings, market trends, and local opportunities. Store user location data and tag alerts and content by region so that users only receive relevant information.

Expand the marketplace feature to better connect farmers with buyers and sellers. Allow users to list crops, livestock, and farm products for sale, and to browse and purchase inputs such as seeds or fertilizer. Enhance this with AI-driven suggestions, such as recommended pricing based on local trends and alerts for optimal selling times. Ensure marketplace listings are filtered and prioritized based on location.

Extend the community system to support expert interactions. Allow users to post questions, optionally attaching images or AI scan results. Add a mechanism to tag posts as “expert requests,” and introduce a way to identify verified experts. AI should assist by summarizing posts and suggesting possible answers, but human experts should remain part of the loop to ensure reliability.

Design and implement a set of planning and calculation tools that are directly useful to farmers. These should include a fertilizer calculator, livestock feed planner, yield estimator, and profit calculator. The tools should accept simple inputs and produce clear outputs, and results should be saved for future reference and integrated into the overall farm management system.

Ensure the system is extensible so that new farm types, such as beekeeping, can be supported without major refactoring. If a farm type is not fully supported, the system should fall back to a generic structure while still allowing data tracking and AI assistance. Over time, specialized modules can be added for these niche types.

All new features must integrate with the existing Supabase backend, use React Query for data fetching, and follow the current project structure. Uploaded media should be stored in Supabase Storage. AI processing and external API calls should be handled via a backend layer or Supabase Edge Functions.

The final result should feel like an intelligent farm assistant that continuously learns from user data, provides timely and actionable insights, generates useful reminders, adapts to different types of farming, and helps farmers improve productivity and decision-making over time.
